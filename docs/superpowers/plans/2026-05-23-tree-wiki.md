# Tree Wiki Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a multi-way tree wiki structure where the root is user-named, non-leaf nodes have names and descriptions, and leaf references point to original document locations using path, line, and limit.

**Architecture:** Add a new deterministic `tree_wiki` module that reads converted Markdown pages from `docs/<source-id>/pages`, extracts section references, deduplicates references, groups them into a broad tree, splits crowded nodes, merges duplicate sibling nodes, and writes `tree.json`, `tree.md`, and `nodes/*.md`. Expose this through a `docs-claw build-tree` CLI command while leaving the existing `build-wiki` behavior intact.

**Tech Stack:** Python standard library dataclasses/json/re/pathlib, existing docs-claw Markdown/frontmatter conventions, pytest.
